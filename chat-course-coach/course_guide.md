# Course Design Guide

## Goal

Convert a user's plain-language learning request into a complete teaching path.

## Rules

- Infer the scope from the user's description.
- Build the outline before teaching.
- Start with fundamentals.
- Break large topics into modules.
- Break modules into lesson-sized pieces.
- Each lesson should contain:
  - core idea
  - plain-language explanation
  - example
  - common mistake
  - quick check

## Mastery Rule

Do not move on until the current lesson is understood.

## Review Rule

If the user is confused:
- re-explain with a simpler analogy
- reduce the example size
- add a tiny practice prompt

## Output Rule

Teach in a conversational, chat-like style.
Do not dump the entire course unless asked.
