# Module: Python高阶编程

## 1. decorator_essence
**题目：** 装饰器是什么？它的本质是什么？
**标签：** Python高阶编程、装饰器、闭包
**难度：** medium
**频率：** 高频
**参考答案：**
装饰器本质上是一个接收函数并返回新函数的高阶函数，作用是在不修改原函数源码的前提下，为函数动态添加额外能力。常见用途包括日志、权限校验、缓存、重试、性能统计。`@decorator` 本质上等价于 `func = decorator(func)`。很多装饰器实现同时会利用闭包保存原函数引用。

## 2. closure_essence
**题目：** 什么是闭包？闭包形成的条件是什么？
**标签：** Python高阶编程、闭包、作用域
**难度：** medium
**频率：** 高频
**参考答案：**
闭包可以理解为“内部函数引用了外部函数作用域中的变量，并且外部函数已经执行结束，但这个变量仍然被保留下来”。形成闭包通常有三个条件：有嵌套函数、内部函数引用外部变量、外部函数返回内部函数或让内部函数在外部继续被使用。闭包常用于延迟执行、封装状态、实现装饰器。

## 3. iterator_vs_generator
**题目：** 迭代器和生成器有什么区别？
**标签：** Python高阶编程、迭代器、生成器
**难度：** medium
**频率：** 高频
**参考答案：**
迭代器是实现了迭代器协议的对象，核心是 `__iter__()` 和 `__next__()`；生成器是创建迭代器的一种更简洁方式，通常通过 `yield` 定义。生成器本质上也是迭代器，但不用手动实现 `__next__()`，写法更自然，且惰性求值、节省内存。

## 4. context_manager
**题目：** 什么是上下文管理器？`with` 语句背后的原理是什么？
**标签：** Python高阶编程、with、上下文管理器
**难度：** medium
**频率：** 高频
**参考答案：**
上下文管理器用于统一管理资源的进入和退出，比如文件、锁、数据库连接。`with` 背后依赖的是对象实现 `__enter__()` 和 `__exit__()` 方法。进入 `with` 时会调用 `__enter__()`，退出时不管是否异常都会调用 `__exit__()`。它的核心价值是让资源获取和释放更安全、更清晰。

## 5. mutable_default_param
**题目：** 可变类型默认参数为什么容易出问题？怎么规避？
**标签：** Python高阶编程、默认参数、函数
**难度：** medium
**频率：** 高频
**参考答案：**
因为函数默认参数在函数定义时只初始化一次，如果默认值是列表、字典这类可变对象，多次调用会共享同一份对象，导致状态污染。常见规避方式是把默认值设为 `None`，在函数体里再判断并初始化新对象。这是 Python 面试里非常经典的坑点。

## 6. shallow_vs_deep_copy
**题目：** 深拷贝和浅拷贝的核心区别是什么？
**标签：** Python高阶编程、拷贝、对象模型
**难度：** medium
**频率：** 高频
**参考答案：**
浅拷贝只复制最外层对象，内部嵌套对象仍然共享引用；深拷贝会递归复制内部对象，得到完全独立的新对象。对于只有一层的简单对象，两者差异不明显；一旦有嵌套列表、字典，自然就会体现出修改联动与否的区别。

## 7. new_vs_init
**题目：** `__new__` 和 `__init__` 有什么区别？
**标签：** Python高阶编程、面向对象、对象创建
**难度：** medium
**频率：** 高频
**参考答案：**
`__new__` 负责创建对象实例，返回的是实例本身；`__init__` 负责初始化这个实例。调用顺序上先 `__new__`，后 `__init__`。大多数业务代码只重写 `__init__`；只有在控制实例创建过程，比如单例、不可变对象子类化时，才更常见重写 `__new__`。

## 8. super_and_mro
**题目：** `super()` 的作用是什么？MRO 又是什么？
**标签：** Python高阶编程、super、多继承
**难度：** medium
**频率：** 高频
**参考答案：**
`super()` 用来按照 MRO（方法解析顺序）去查找父类或下一个类的方法，而不是简单地“找直接父类”。MRO 是 Python 在多继承场景下确定方法查找顺序的一套规则。理解 `super()` 不能只停留在“调用父类方法”，更准确地说，是按 MRO 链继续往后找。

## 9. property_usage
**题目：** `@property` 有什么作用？适合什么场景？
**标签：** Python高阶编程、property、封装
**难度：** medium
**频率：** 高频
**参考答案：**
`@property` 可以把方法伪装成属性访问，让调用方写起来像在访问字段，但背后仍能执行逻辑。它常用于封装校验、懒加载、计算属性。好处是对外接口更优雅，同时又保留控制能力。

## 10. slots_usage
**题目：** `__slots__` 的作用是什么？
**标签：** Python高阶编程、slots、内存优化
**难度：** medium
**频率：** 中高频
**参考答案：**
`__slots__` 用来限制实例可以绑定的属性集合，并在一定场景下减少内存开销。它常用于大量对象实例、字段固定的场景。代价是灵活性下降，比如不能随意动态加属性，和某些继承场景要一起考虑。

## 11. dunder_methods
**题目：** 常见魔术方法有哪些？实际开发里最常用的是哪几类？
**标签：** Python高阶编程、魔术方法、对象模型
**难度：** medium
**频率：** 中高频
**参考答案：**
常见魔术方法包括对象表示类（`__str__`、`__repr__`）、比较类（`__eq__`、`__lt__`）、容器类（`__len__`、`__getitem__`、`__iter__`）、上下文管理类（`__enter__`、`__exit__`）等。开发中最常用的一般是对象打印、容器行为定制和上下文管理相关方法。

## 12. gil_threads_processes
**题目：** GIL 是什么？它对 Python 线程、进程、协程的理解有什么影响？
**标签：** Python高阶编程、GIL、并发
**难度：** medium
**频率：** 中高频
**参考答案：**
GIL 是 CPython 解释器中的全局解释器锁，同一时刻通常只允许一个线程执行 Python 字节码。它意味着 CPU 密集型任务里，多线程未必能提升性能；这类场景更常考虑多进程。对于 IO 密集型任务，多线程仍然有价值。协程不是靠多核并行，而是通过事件循环在单线程里高效切换等待任务。
