# QUESTION_BANK_GUIDE

本文件定义 `python-testdev-drill` 的全局题库组织规则，而不是只描述某一个模块。

## 一、全局出题原则

所有模块，不管是：
- 刷题模式
- 做题模式

默认都遵守同一条规则：

### 默认优先级
1. 高频
2. 中高频
3. 加分题

### 默认抽题顺序
- 先抽当前模块中**未出过的高频题**
- 高频题出完后，再抽**未出过的中高频题**
- 中高频题出完后，再抽**未出过的加分题**
- 若系统题不足，再考虑未出过的自建题
- `刷题模式` 和 `做题模式` 共用同一套去重记录，不能因为模式切换而重复出旧题
- 若题池耗尽，必须明确提示用户是否允许重复

## 二、特殊模式规则

### 1. 错题模式
- 默认从该模块错题集里抽题
- 如果错题本身带有频率信息，仍优先抽高频错题

### 2. 收藏模式
- 默认从该模块收藏题库里抽题
- 如果收藏题本身带有频率信息，仍优先抽高频收藏题

### 3. 自建题库模式
- 默认从该模块自建题库里抽题
- 如果自建题带有频率信息，仍优先抽高频自建题

## 三、当前模块建议覆盖方向

### 1. Python基础手写代码题
建议优先覆盖：
- 基础算法与数据结构
- Python 高级特性应用
- 自动化测试核心组件封装
- 数据库与数据校验
- UI 自动化封装
- 工程化与辅助脚本

### 2. Python编程基础理论面试题
建议优先覆盖：
- 数据类型
- 深浅拷贝
- 装饰器
- 生成器 / 迭代器
- 闭包
- GIL
- 协程
- 异常
- 面向对象

### 3. 自动化测试题
建议优先覆盖：
- 用例设计
- 接口测试
- UI 自动化
- Mock
- Pytest
- CI/CD
- 稳定性治理
- 环境隔离
- 质量指标

### 4. Pytest框架
建议优先覆盖：
- fixture
- conftest.py
- 参数化
- mark
- setup / teardown 与 fixture 区别
- skip / skipif / xfail
- pytest.ini
- Hook 机制基础
- 用例组织与收集规则
- 常见插件与报告机制

### 5. 测试开发面试题
建议优先覆盖：
- 平台设计
- 调度系统
- 报告系统
- 权限设计
- 可观测性
- 异步与并发
- 数据建模
- 服务设计

### 6. 测试工程化与CI/CD
建议优先覆盖：
- Git
- Jenkins
- Docker
- Linux
- SQL
- 持续集成
- 发布流程
- 环境管理
- 日志排查
- 数据校验

## 四、未来新模块规则

以后新增任何模块，都必须自动继承这条规则：

### 新模块建库顺序
1. 先分析该模块在真实面试中的**高频题型**
2. 先建立高频题层
3. 再建立中高频题层
4. 最后补加分题层

### 新模块出题规则
- 新模块默认优先抽高频题
- 只有高频题刷完后，才进入中高频题
- 只有中高频题刷完后，才进入加分题

## 五、题目结构建议

每道题尽量包含：
- id / 题号
- 标题
- 题目正文
- 标签（中文）
- 难度
- 频率（高频 / 中高频 / 加分题）
- 参考答案

## 六、输出风格

- 标签统一使用中文
- 可在题目中标出频率，例如：`频率：高频`
- 面试答案统一使用：
  - 口语化表达
  - 专业
  - 系统
  - 逻辑清晰
  - 完整回答风格
